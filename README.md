# QMS 系统选型评估（qms-selection）

> 面向 IT/质量信息化负责人的混合式双版技能。对质量管理体系（QMS）软件做结构化选型评估，明确需求与评分维度，输出选型评估矩阵与推荐报告（Markdown + HTML 双版）。

## 适用角色

- IT 负责人、质量信息化负责人
- 质量总监 / 体系经理（需求方）

## 核心能力

1. 澄清业务需求与**合规框架**（IATF 16949 / ISO 9001 / ISO 17025 等）。
2. 定义**评分维度与权重**（功能覆盖 / 合规 / 架构集成 / 易用性 / 实施服务 / TCO / 扩展）。
3. 多候选系统**加权评分矩阵**（1–5 分制，权重合计=1）。
4. 排序并形成短名单与**推荐结论**。
5. 一键产出双版本：`QMS选型评估报告.md` 与 `QMS选型评估报告.html`（主色 #C8102E）。

## 目录结构

```
qms-selection/
├── SKILL.md
├── README.md
├── references/
└── scripts/
    └── build_report.py     # 双版报告生成器（含内置小样本）
```

## 快速使用

```bash
python scripts/build_report.py --demo
python scripts/build_report.py --input result.json \
    --md-out QMS选型评估报告.md --html-out QMS选型评估报告.html
```

## 评分方法

- 各维度 1–5 分；加权总分 = Σ(权重 × 分值)。
- 权重合计须为 1；脚本会自动校验，若合计≠1 则归一化到 5 分制并提示。

## 防幻觉声明

- 不编造供应商报价、功能清单、合规认证；缺失标注「待企业补充」。
- 仅提供评估矩阵与推荐建议，不做最终采购决策。
